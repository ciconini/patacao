# Modules

Feature-based modules following domain-driven design principles.

Each module contains:
- **pages/**: Route components (list, detail, create/edit pages)
- **components/**: Module-specific components
- **hooks/**: Module-specific hooks and state management
- **api/**: API service wrappers for module endpoints
- **tests/**: Module-specific tests

## Modules

- **administrative/**: Company, stores, customers, pets, users management
- **services/**: Services catalog, appointments, calendar
- **financial/**: POS, transactions, invoices, credit notes
- **inventory/**: Products, suppliers, stock management, purchase orders
- **users/**: User access management, roles, permissions
